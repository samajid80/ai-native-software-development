# Lesson 3: Multiple Choices with Elif

## Opening Hook

Grading depends on score: IF score >= 90, grade is A. ELIF score >= 80, grade is B. ELIF score >= 70, grade is C. ELSE, grade is F. What if you wrote this as multiple if statements instead? All of them would test, even after you found a match. The **elif** statement solves thisâ€”it tests conditions IN ORDER until one is true, then stops. This lesson teaches elif chains for multiple choices.

## Elif Syntax: Testing Multiple Conditions in Sequence

**elif** stands for "else-if". It lets you test another condition if the previous one was false.

**Pattern:**

```python
if condition1:
    # runs if condition1 is True
elif condition2:
    # runs if condition1 is False AND condition2 is True
elif condition3:
    # runs if condition1 and condition2 are False AND condition3 is True
else:
    # runs if ALL conditions are False
```

**Critical rule:** Conditions are tested in order, top to bottom. The FIRST condition that is true runs, and all others are skipped.

**Worked example: Temperature categories**

```python
temp: int = 18
if temp < 0:
    print("Freezing")
elif temp < 15:
    print("Cold")
elif temp < 25:
    print("Mild")
else:
    print("Hot")

# Trace: temp=18
# Is 18 < 0? No â†’ skip if block
# Is 18 < 15? No â†’ skip elif block
# Is 18 < 25? Yes â†’ prints "Mild", stop checking
```

## Order Matters: Why Condition Sequence Is Critical

The ORDER of elif conditions is critical. Test broader conditions LAST; test specific conditions FIRST.

**BAD ORDER (Wrong logic):**

```python
score: int = 95
if score >= 70:
    print("Pass")  # This runs for ANY score >= 70
elif score >= 90:
    print("Excellent")  # This NEVER runs for 95 (already passed the first condition)
```

**CORRECT ORDER (Right logic):**

```python
score: int = 95
if score >= 90:
    print("Excellent")  # Test most specific first (90+)
elif score >= 80:
    print("Good")  # Then less specific (80-89)
elif score >= 70:
    print("Pass")  # Then least specific (70-79)
else:
    print("Fail")  # Finally, the fallback
```

## Reading and Tracing Elif Chains

To predict elif output, trace it:
1. Test first condition
2. If true, run that block and STOP
3. If false, test next condition
4. Repeat until one is true or reach else
5. Only ONE block ever runs

**Worked example: Trace with different inputs**

```python
age: int = 25
if age < 13:
    print("Child")
elif age < 18:
    print("Teen")
elif age < 65:
    print("Adult")
else:
    print("Senior")
```

**Trace with age = 25:**
- Is 25 < 13? No
- Is 25 < 18? No
- Is 25 < 65? Yes â†’ prints "Adult", stop

**Trace with age = 10:**
- Is 10 < 13? Yes â†’ prints "Child", stop

**Trace with age = 70:**
- Is 70 < 13? No
- Is 70 < 18? No
- Is 70 < 65? No â†’ prints "Senior" (else block)

## Common Mistakes in Elif Chains

**Mistake 1: Using `=` instead of `==`**

```python
# WRONG - = is assignment, not comparison
if age = 18:  # SyntaxError
    print("Adult")

# CORRECT - == is comparison
if age == 18:  # Checks if age equals 18
    print("Adult")
```

**Mistake 2: Forgetting indentation**

```python
# WRONG
if age >= 18:
print("Adult")  # IndentationError

# CORRECT
if age >= 18:
    print("Adult")
```

**Mistake 3: Assuming multiple if statements act like elif**

```python
# WRONG THINKING - both blocks might run
score: int = 95
if score >= 90:
    print("A")
if score >= 80:  # This ALSO runs because it's a separate if
    print("B")
# Prints: A, B (both run!)

# CORRECT - elif stops after first match
if score >= 90:
    print("A")
elif score >= 80:  # This doesn't test if first was true
    print("B")
# Prints: A (only one runs)
```
